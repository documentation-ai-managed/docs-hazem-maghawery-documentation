## Creating Your First Document

<Callout kind="info">Documents support multiple formats including Markdown, rich text, and structured templates.</Callout>

<Tabs>
  <Tab title="From Template" icon="file-text">
    Choose from pre-built templates for common documentation types like API guides, user manuals, or project plans.
    
    <CodeGroup tabs="Markdown,Rich Text">
      ````markdown
      # API Documentation
      
      ## Authentication
      
      Use your API key to authenticate requests:
      
      ```javascript
      const response = await fetch('/api/data', {
        headers: { 'Authorization': `Bearer ${API_KEY}` }
      });
      ```
      ````
      ````javascript
      // Rich text content would be edited in the visual editor
      const richContent = {
        type: 'doc',
        content: [
          { type: 'heading', attrs: { level: 1 }, content: 'API Documentation' },
          { type: 'heading', attrs: { level: 2 }, content: 'Authentication' },
          { type: 'paragraph', content: 'Use your API key...' }
        ]
      };
      ````
    </CodeGroup>
  </Tab>
  <Tab title="Import Existing" icon="upload">
    Import documents from external sources like GitHub repositories, Confluence spaces, or local files.
    
    <Expandable title="Supported Formats" default-open="false">
      - Markdown files (.md)
      - HTML documents
      - PDF files (converted to editable format)
      - JSON structured content
    </Expandable>
  </Tab>
</Tabs>

## Best Practices

Use consistent naming conventions for documents and folders. Regularly review and update outdated content. Leverage templates for common document types to maintain consistency across your organization.

Set up automated backups and consider using the search functionality to quickly find relevant information.

For larger teams, establish guidelines for document ownership and review processes to ensure quality and accuracy.

## Troubleshooting Common Issues

<Expandable title="Login Problems" default-open="false">
  If you can't log in, check that you've verified your email address and that your password meets the minimum requirements.
</Expandable>

<Expandable title="Import Failures" default-open="false">
  Ensure your import files are in supported formats and don't exceed size limits. Check for special characters that might cause parsing issues.
