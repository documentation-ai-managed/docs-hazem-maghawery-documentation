## Basic Settings

<Callout kind="info">Access settings from the user menu in the top-right corner of the interface.</Callout>

<Tabs>
  <Tab title="Profile" icon="user">
    Update your personal information, notification preferences, and display options.
    
    <CodeGroup tabs="API,UI">
      ````javascript
      // Update profile via API
      const profileUpdate = {
        name: 'John Doe',
        email: 'john@example.com',
        notifications: {
          email: true,
          inApp: false
        }
      };
      await fetch('/api/user/profile', {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(profileUpdate)
      });
      ````
      ````javascript
      // In the UI, navigate to Settings > Profile
      // Update fields and save changes
      const updatedProfile = {
        displayName: 'John Doe',
        timezone: 'America/New_York',
        theme: 'dark'
      };
      ````
    </CodeGroup>
  </Tab>
  <Tab title="Security" icon="shield">
    Manage password, two-factor authentication, and API key settings.
    
    <Steps>
      <Step title="Enable 2FA" icon="lock">
        Go to Security settings and follow the setup wizard for two-factor authentication.
      </Step>
      <Step title="Generate API Key" icon="key">
        Create API keys with specific permissions for different applications.
      </Step>
      <Step title="Review Sessions" icon="eye">
        Check active sessions and revoke access from unrecognized devices.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Document Templates

<Tabs>
  <Tab title="Create Template" icon="file-plus">
    Design custom templates for common document types used in your organization.
    
    <CodeGroup tabs="JSON,YAML">
      ````json
      {
        "name": "API Documentation Template",
        "fields": [
          { "name": "endpoint", "type": "string", "required": true },
          { "name": "method", "type": "enum", "values": ["GET", "POST", "PUT", "DELETE"] },
          { "name": "description", "type": "text", "required": true }
        ],
        "structure": {
          "sections": ["Overview", "Authentication", "Examples"]
        }
      }
      ````
      ````yaml
      name: API Documentation Template
      fields:
        - name: endpoint
          type: string
          required: true
        - name: method
          type: enum
          values: [GET, POST, PUT, DELETE]
        - name: description
          type: text
          required: true
      structure:
        sections: [Overview, Authentication, Examples]
      ````
    </CodeGroup>
  </Tab>
  <Tab title="Apply Templates" icon="check-circle">
    Apply templates to new documents for consistent structure and formatting.
  </Tab>
</Tabs>

## Performance Tuning

For large documentation spaces, optimize performance by enabling caching, compressing images, and limiting concurrent users per document.

Use the analytics dashboard to monitor usage patterns and identify bottlenecks in your configuration.
